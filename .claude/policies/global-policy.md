# Global Policy — bkit-doctor

## Core Rules

1. **Phase 단위 운영** — 모든 작업은 phase 단위로 관리한다.
2. **Plan/Design 선행** — 코드 작성 전 Plan 또는 Design 문서가 승인되어 있어야 한다.
3. **문서 없는 구현 금지** — 문서 없이 implementer를 투입하지 않는다.
4. **변경 이유 기록 필수** — 모든 구현 변경은 Why를 문서에 남긴다.
5. **Planner 우선** — 모든 작업 요청은 planner-orchestrator를 먼저 통과한다.
6. **작업은 멈추지 않는다** — 막히면: 현황 요약 → 대안 제시 → 안전한 다음 행동으로 계속.
7. **자동 배포 금지** — 배포는 수동 승인 후에만.
8. **코드 유출 금지** — 프로젝트 외부로 소스 코드 전송 금지.
9. **팀 모드 항상** — 단독 처리보다 역할별 agent 분리 우선.
10. **컨텍스트 절약** — 응답은 짧고 핵심만.

## Blocked State Protocol
막혔을 때 이 순서로 처리:
1. 현재 상태 4줄 요약 출력
2. 대안 2가지 이상 제시
3. 가장 안전한 대안 선택 후 계속 진행
4. 사용자 확인이 필요한 경우에만 중단
